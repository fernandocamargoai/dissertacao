\section{Descrição dos Algoritmos Avaliados}
\label{cap:sistemasHidrotermicos} 

Foram implementados alguns dos mais eficientes algoritmos de otimização multiobjetivo encontrados na literatura atual na solução do Problema do Planejamento Energético de Sistemas Hidrotérmicos. Três deles são baseados em algoritmos evolucionários: NSGA-II \cite{deb2002}, SPEA2 \cite{zitzler2001} e MOCell \cite{nebro2006} \cite{nebro2007}; e dois são baseados em enxame de partículas: OMOPSO \cite{reyes2005} e SMPSO \cite{nebro2009}.

NSGA-II é um algoritmo genético baseado na obtenção de uma nova população da população original através da aplicação dos operadores genéticos (seleção, cruzamento e mutação). Entretanto, ao invés de passar essa nova população diretamente para a próxima iteração, ela é combinada com a anterior e as soluções são ordenadas por um rank. Assim, apenas as melhores soluções dessa combinação são escolhidas para gerar uma nova população. Caso aconteça algum empate na avaliação do rank, uma estimativa de densidade baseada em crowding distance dos individuos ao redor é usada para selecionar as soluções mais promissoras.

SPEA2 também é um algoritmo genético. Porém, nesse algoritmo, cada indivíduo tem um valor de fitness calculado pela soma de seu fitness puro com uma estimação de densidade. O algoritmo aplica os operadoradores genéticos para preencher um arquivo de indivíduos. Então, os indivíduos não dominados da combinação da população original com o arquivo são copiados para uma nova população. Se o número de indivíduos não dominados for maior do que o tamanho da população, um operadora de truncamento baseado no cálculo de KNN (K-th Nearest Neighbor) é usado. Dessa forma, os indíviduos com menores distâncias qualquer outro indivíduo são selecionados.

MOCell é um algoritmo genético celular (cGA). Ele inclui um arquivo externo para guardar soluções não dominadas encontradas durante sua execução. Esse arquivo faz uso de crowding distance do NSGA-II para manter a diversidade.

OMOPSO é um algoritmo multiobjetivo de enxame de partículas. Assim como o MOCell, ele faz uso de um arquivo externo para guardar soluções não dominadas encontradas durante sua execução, também com crowding distance do NSGA-II para filtrar as melhores soluções. Além disso, diferente de outros algoritmos de enxame de partículas, ele faz uso de mutação para acelerar a convergência do enxame. Seu arquivo externo faz uso do conceito de dominância $\epsilon$ para limitar o número de soluções guardadas.

SMPSO é um variante do OMOPSO com a adoção de uma nova abordagem de constrição de velocidade com o fim de limitar a velocidade das partículas. Esse constrição evita que a velocidade das partículasalcance valores altos demais e pequenos demais, o que pode prejudicar a busca.
